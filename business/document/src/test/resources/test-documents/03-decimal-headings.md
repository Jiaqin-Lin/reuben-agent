1 系统架构设计

本节概述整个系统的架构设计理念和分层策略。

1.1 整体架构

系统采用分层架构，自上而下分为接入层、业务层、基础设施层。

1.1.1 接入层设计

接入层负责请求路由、认证鉴权、限流熔断。使用 Spring Cloud Gateway 作为统一入口。

1.1.2 业务层设计

业务层按领域划分为多个微服务，每个微服务独立部署、独立扩展。

1.2 数据架构

数据存储采用混合持久化策略，关系型数据使用 MySQL，图数据使用 Neo4j。

1.2.1 MySQL 分库分表策略

按租户 ID 进行水平分库，按时间维度进行分区。

1.2.2 Neo4j 图建模

知识图谱节点包括文档节点、概念节点、实体节点，边表示包含、引用、关联等关系。

1.3 部署架构

使用 Kubernetes 作为容器编排平台，每个微服务至少 3 个副本。

2 核心模块设计

以下各节详细介绍各个核心模块的设计方案。

2.1 Document 模块

负责文档的上传、解析、结构提取和存储。

2.1.1 上传流程

文件上传后先经过 Tika 文本提取，再进行结构化分析。

2.1.2 结构提取管线

原文件 → Tika 提取纯文本 → 信号提取 → 结构节点归并 → 索引存储。

2.2 Agent 模块

智能体模块负责接收用户指令并通过推理引擎执行任务。

3 接口规范

3.1 RESTful API 设计

所有 API 遵循 REST 风格，统一返回 ApiResponse 格式。

3.1.1 成功响应格式

成功时返回 code=0，data 携带业务数据。

3.1.2 错误响应格式

错误时返回对应的错误码和消息，由 GlobalExceptionHandler 统一处理。

3.2 认证与鉴权

使用 Spring Security + JWT 实现无状态认证。

这是一段正文，介绍本文档的目的。本文档用于测试 Markdown 各级标题的识别能力。

## 1.1 项目背景

随着企业数字化转型的深入，文档管理成为知识管理的核心环节。本系统提供从文档上传到结构提取的全流程自动化。

### 1.1.1 现有问题

传统文档管理存在三大问题：分类靠人工、检索效率低、知识碎片化。

### 1.1.2 解决方案

本系统采用结构化提取 + 语义索引的双引擎架构。

## 1.2 技术选型

关键技术栈包括 Spring Boot、MyBatis-Plus、Elasticsearch 和 Neo4j。

# 系统部署操作指南

以下是部署系统的详细操作步骤，请严格按顺序执行。

## 环境准备阶段

在进行部署之前，需要完成以下准备工作。

第1步 检查服务器配置

确认目标服务器满足最低配置要求：4 核 CPU、16 GB 内存、100 GB 可用磁盘空间。

第2步 安装依赖软件

需要在服务器上预先安装 Docker 24+ 和 docker-compose 2+。

第3步 配置环境变量

复制 .env.example 为 .env，并根据实际环境修改数据库密码、密钥等敏感配置。

## 部署执行阶段

Step 1 拉取镜像

执行 docker-compose pull 命令拉取所有服务的 Docker 镜像。这一步可能需要几分钟，取决于网络速度。

Step 2 启动中间件

先启动 MySQL、Redis、Elasticsearch 等基础中间件，等待健康检查通过后再启动业务服务。

步骤三 初始化数据库

运行 SQL 迁移脚本创建表结构和初始数据。

STEP 4 启动业务服务

依次启动 launcher 模块，确认所有服务注册到 Nacos 并健康。

第5步 验证部署结果

访问健康检查端点 /actuator/health，确认所有组件状态为 UP。

## 其他操作

除了部署之外，日常运维中还需要掌握以下操作。

(1) 查看日志：使用 docker-compose logs -f 查看实时日志

(2) 重启服务：使用 docker-compose restart <服务名> 重启指定服务

（3）数据备份：定期执行数据库备份脚本

（a）全量备份：每天凌晨执行一次

（b）增量备份：每小时执行一次
